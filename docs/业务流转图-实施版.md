### 2.2 核心表结构（简化）

**用户表 (users)**
```sql
CREATE TABLE users (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  openid VARCHAR(64) NOT NULL COMMENT '小程序openid',
  unionid VARCHAR(64) COMMENT '微信unionid',
  phone VARCHAR(20),
  nickname VARCHAR(64),
  avatar VARCHAR(256),
  source VARCHAR(32) COMMENT '来源: store/referral/activity',
  referrer_id BIGINT COMMENT '推荐人ID',
  store_id BIGINT COMMENT '归属门店',
  staff_id BIGINT COMMENT '归属导购',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_openid (openid),
  INDEX idx_phone (phone)
);
```

**商品表 (products)**
```sql
CREATE TABLE products (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(128) NOT NULL,
  description TEXT,
  category_id BIGINT,
  price DECIMAL(10,2) NOT NULL,
  original_price DECIMAL(10,2),
  main_image VARCHAR(256),
  images JSON,
  status TINYINT DEFAULT 0 COMMENT '0:draft, 1:online, 2:offline',
  supplier_id BIGINT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_status (status),
  INDEX idx_category (category_id)
);
```

**SKU表 (product_skus)**
```sql
CREATE TABLE product_skus (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  product_id BIGINT NOT NULL,
  specs JSON COMMENT '{"color": "黑色", "size": "M"}',
  price DECIMAL(10,2) NOT NULL,
  stock INT DEFAULT 0,
  sku_code VARCHAR(64),
  image VARCHAR(256),
  INDEX idx_product (product_id)
);
```

**订单表 (orders)**
```sql
CREATE TABLE orders (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_no VARCHAR(32) NOT NULL UNIQUE,
  user_id BIGINT NOT NULL,
  status TINYINT DEFAULT 0 COMMENT '订单状态',
  total_price DECIMAL(10,2) NOT NULL,
  pay_price DECIMAL(10,2) NOT NULL,
  discount_price DECIMAL(10,2) DEFAULT 0,
  delivery_type TINYINT COMMENT '1:快递 2:自提',
  address_snapshot JSON,
  pay_time DATETIME,
  ship_time DATETIME,
  receive_time DATETIME,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_user (user_id),
  INDEX idx_status (status),
  INDEX idx_order_no (order_no)
);
```

**售后表 (after_sales)**
```sql
CREATE TABLE after_sales (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_id BIGINT NOT NULL,
  order_item_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  type TINYINT COMMENT '1:仅退款 2:退货退款 3:换货',
  status TINYINT DEFAULT 0,
  reason VARCHAR(256),
  refund_amount DECIMAL(10,2),
  tracking_no VARCHAR(64),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_order (order_id),
  INDEX idx_user (user_id)
);
```

## 三、核心状态机

### 3.1 订单状态机

```
                    ┌─────────────────────────────────────┐
                    │                                     │
                    ▼                                     │
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 待支付   │───→│ 已支付   │───→│ 已发货   │───→│ 已完成   │
│ PENDING  │    │ PAID     │    │ SHIPPED  │    │ COMPLETED│
└──────────┘    └──────────┘    └──────────┘    └──────────┘
     │               │               │
     │ 超时/取消      │               │
     ▼               ▼               ▼
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 已取消   │    │ 售后中   │    │ 售后中   │
│ CANCELED │    │ REFUNDING│    │ REFUNDING│
└──────────┘    └──────────┘    └──────────┘
                     │
                     │ 退款完成
                     ▼
                ┌──────────┐
                │ 已退款   │
                │ REFUNDED │
                └──────────┘
```

**状态转换触发条件**：

| 当前状态 | 触发事件 | 目标状态 |
|----------|----------|----------|
| PENDING | 支付成功 | PAID |
| PENDING | 超时/用户取消 | CANCELED |
| PAID | 运营发货 | SHIPPED |
| PAID | 用户申请售后 | REFUNDING |
| SHIPPED | 用户确认收货 | COMPLETED |
| SHIPPED | 用户申请售后 | REFUNDING |
| REFUNDING | 退款完成 | REFUNDED |

### 3.2 售后状态机

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 待审核   │───→│ 待退货   │───→│ 待确认   │───→│ 已完成   │
│ PENDING  │    │ RETURNING│    │ CONFIRMING│   │ COMPLETED│
└──────────┘    └──────────┘    └──────────┘    └──────────┘
     │
     │ 审核拒绝
     ▼
┌──────────┐
│ 已拒绝   │
│ REJECTED │
└──────────┘
```

## 五、关键业务流程（技术视角）

### 5.1 用户注册与登录

```
小程序调用 wx.login()
    │
    ↓
前端获取 code
    │
    ↓
POST /api/user/login { code, encryptedData, iv }
    │
    ↓
后端用 code 换 openid/session_key
    │
    ↓
解密获取手机号（可选）
    │
    ↓
查询/创建用户记录
    │
    ↓
生成 JWT token 返回
```

### 5.2 下单支付流程

```
POST /api/orders { items, address_id, delivery_type }
    │
    ├── 校验库存（Redis 锁）
    ├── 计算价格（商品价 + 运费 - 优惠）
    ├── 创建订单（状态: PENDING）
    └── 返回订单号
    │
    ↓
POST /api/orders/:id/pay
    │
    ├── 调用微信支付统一下单
    ├── 返回支付参数
    └── 前端调用 wx.requestPayment()
    │
    ↓
微信支付回调
    │
    ├── 验签
    ├── 更新订单状态为 PAID
    ├── 扣减库存
    ├── 发送订单通知（企微/订阅消息）
    └── 返回成功
```

### 5.3 发货流程

```
POST /admin/orders/:id/ship { tracking_company, tracking_no }
    │
    ├── 校验订单状态为 PAID
    ├── 更新订单状态为 SHIPPED
    ├── 记录物流信息
    └── 发送发货通知（订阅消息）
```

### 5.4 售后退款流程

```
POST /api/after-sales { order_id, type, reason, images }
    │
    ├── 校验订单状态（PAID/SHIPPED/COMPLETED）
    ├── 创建售后单（状态: PENDING）
    └── 通知运营
    │
    ↓
运营审核（POST /admin/after-sales/:id/approve 或 reject）
    │
    ├── approve: 状态 → RETURNING，提供退货地址
    └── reject: 状态 → REJECTED，填写原因
    │
    ↓
用户填写退货物流（PUT /api/after-sales/:id/tracking）
    │
    ├── 状态 → CONFIRMING
    └── 通知运营
    │
    ↓
运营确认收货并退款（POST /admin/after-sales/:id/refund）
    │
    ├── 调用微信退款接口
    ├── 退款成功后更新状态 → COMPLETED
    ├── 恢复库存（如需）
    └── 通知用户
```
