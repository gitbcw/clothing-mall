### 2.2 Node.js 兼容性配置

Windows 下 Vue 2 项目需要设置环境变量：

```bash
# 临时设置（每次终端需要执行）
export NODE_OPTIONS=--openssl-legacy-provider

# 或在 PowerShell 中
$env:NODE_OPTIONS="--openssl-legacy-provider"
```

## 三、数据库配置

### 3.1 Docker 方式（推荐）

```bash
cd docker

# 仅启动 MySQL
docker compose up mysql -d

# 查看状态
docker compose ps

# 查看日志
docker compose logs mysql
```

**连接信息：**

| 参数 | 值 |
|------|------|
| Host | localhost 或 127.0.0.1 |
| Port | 3306 |
| Database | clothing_mall |
| Username | clothing_mall |
| Password | clothing123456 |
| Root Password | root123456 |

### 3.2 初始化数据库

```bash
# 如果是首次运行，需要导入初始数据
cd docker
docker compose up mysql -d

# 等待 MySQL 启动完成（约 10 秒）
# 初始化 SQL 会自动执行（docker/db/init-sql/）

# 或手动导入
mysql -h 127.0.0.1 -u clothing_mall -pclothing123456 clothing_mall < docker/db/init-sql/clothing_mall.sql
```

### 3.3 本地安装 MySQL（备选）

如果不想用 Docker，可以本地安装 MySQL 8.0，创建数据库和用户：

```sql
CREATE DATABASE clothing_mall CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'clothing_mall'@'localhost' IDENTIFIED BY 'clothing123456';
GRANT ALL PRIVILEGES ON clothing_mall.* TO 'clothing_mall'@'localhost';
FLUSH PRIVILEGES;
```

## 四、后端启动

### 4.1 首次构建

```bash
cd C:/MyFile/workspace/clothing-mall

# Maven 构建（首次或依赖变更时）
mvn clean install -DskipTests
```

### 4.2 启动聚合服务（推荐）

聚合服务包含所有 API，端口 8080：

```bash
cd C:/MyFile/workspace/clothing-mall

# 方式一：直接运行 jar
java -Dfile.encoding=UTF-8 -jar clothing-mall-all/target/clothing-mall-all-0.1.0-exec.jar

# 方式二：指定配置（如需要）
java -Dfile.encoding=UTF-8 -Dspring.profiles.active=db,core,admin,wx -jar clothing-mall-all/target/clothing-mall-all-0.1.0-exec.jar
```

### 4.3 启动独立服务（可选）

```bash
# 仅启动 Admin API（端口 8083）
java -Dfile.encoding=UTF-8 -jar clothing-mall-admin-api/target/clothing-mall-admin-api-0.1.0-exec.jar

# 仅启动 WX API（端口 8082）
java -Dfile.encoding=UTF-8 -jar clothing-mall-wx-api/target/clothing-mall-wx-api-0.1.0-exec.jar
```

### 4.4 验证后端

```bash
# 健康检查
curl http://localhost:8080/admin/auth/info

# Swagger API 文档
# 浏览器打开：http://localhost:8080/swagger-ui.html
```

## 八、常见问题

### 8.1 后端启动失败

**问题**：数据库连接失败
```
解决方案：
1. 确认 MySQL 容器已启动：docker compose ps
2. 确认数据库存在：mysql -u clothing_mall -p
3. 检查配置：clothing-mall-db/src/main/resources/application-db.yml
```

**问题**：端口被占用
```bash
# Windows 查找占用进程
netstat -ano | findstr :8080

# 结束进程（替换 PID）
taskkill /PID <PID> /F
```

### 8.2 前端启动失败

**问题**：`ERR_OSSL_EVP_UNSUPPORTED`
```bash
# 设置 Node 选项
export NODE_OPTIONS=--openssl-legacy-provider
npm run dev
```

**问题**：依赖安装失败
```bash
# 清理缓存重试
rm -rf node_modules package-lock.json
npm cache clean --force
npm install
```

### 8.3 Docker 问题

**问题**：MySQL 容器无法启动
```bash
# 查看日志
docker compose logs mysql

# 重置（会删除数据）
docker compose down -v
docker compose up mysql -d
```

## 九、开发注意事项

### 9.1 本地联调

- 优先使用 `127.0.0.1` 而非 `localhost`（某些场景下 DNS 解析问题）
- 前端代理已配置，直接请求 `/admin/*` 和 `/wx/*` 即可

### 9.2 数据库变更

- 开发环境数据库：`clothing_mall`
- 测试环境数据库：`clothing_mall_test`（如有）
- **不要在生产数据库上直接测试**

### 9.3 热重载

- **前端**：`npm run dev` 自动热重载
- **后端**：默认关闭（`spring.devtools.restart.enabled: false`）
  - 如需开启，修改 `application.yml` 并添加 spring-boot-devtools 依赖
