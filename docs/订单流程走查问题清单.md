# 订单流程走查问题清单

> **日期**：2026-03-19
> **走查范围**：管理后台 + 小程序
> **核心背景**：川着 transmute 采用「只换不退」模式，需全面梳理相关功能

---

## 一、订单模块问题

### 1.1 售后按钮文案与流程不符

| 问题 | 说明 | 优先级 |
|------|------|--------|
| 申请退款按钮 | 应改为「申请换货」，因为只换不退 | P0 |
| 已退款状态 | 应改为「已换货」 | P0 |
| 退款按钮存在 | 列表中不应出现退款按钮，应是换货入口 | P0 |

**需确认**：只换不退的法律合规性（电商法规是否支持）

### 1.2 订单列表按钮状态不联动

| 问题 | 说明 | 优先级 |
|------|------|--------|
| 筛选与按钮不匹配 | 在「已发货」筛选下，仍显示「发货」按钮 | P1 |
| 角标未动态更新 | 已发货后，「待发货」角标仍显示原数量 | P1 |

### 1.3 发货功能缺少自提选项

| 问题 | 说明 | 优先级 |
|------|------|--------|
| 仅支持快递 | 发货仅能选择快递公司、填写快递单号 | P1 |
| 缺少到店自提 | 未支持到店自提的发货方式 | P1 |

**设计要求**：
- 自提：无需填写快递单号
- 自提：生成核销码供顾客到店使用

### 1.4 换货流程未实现

| 问题 | 说明 | 优先级 |
|------|------|--------|
| 无换货入口 | 小程序端缺少换货申请入口 | P0 |
| 差价处理 | 换货涉及差价（补价/退价）如何处理？ | P0 |
| 换货规格确认 | 需确认换货商品的规格（颜色/尺码） | P0 |
| 换货通知 | 顾客收到换货进度通知 | P1 |

**换货流程（需实现）**：
```
顾客申请换货 → 选择新规格 → 确认差价 → 审核通过 →
顾客寄回/到店 → 验货 → 发出新商品 → 换货完成
```

### 1.5 退款特殊通道

| 问题 | 说明 | 优先级 |
|------|------|--------|
| 特殊情况退款 | 客户坚持退款时需留口子 | P1 |

**建议**：运营后台人工判断是否允许退款（非自助）

## 二、优惠券模块问题

### 2.1 优惠券发放流程未打通

| 问题 | 说明 | 优先级 |
|------|------|--------|
| 创建后如何发放 | 后台创建优惠券后，如何到用户手中？ | P0 |
| 缺少领券入口 | 小程序端用户从何处领券？ | P0 |
| 注册发券 | 新用户注册时自动发放优惠券，逻辑未实现 | P0 |
| 优惠券使用 | 用户在结算时如何使用优惠券？ | P0 |

**需确认的流程**：
1. 后台创建优惠券（面额、门槛、生效/失效时间）
2. 优惠券发放方式：
   - 手动发放（后台指定用户）
   - 自动发放（注册、满减触发）
   - 用户自领（小程序领券中心）
3. 结算时如何使用（自动匹配/用户选择）

### 2.2 小程序端优惠券展示

| 问题 | 说明 | 优先级 |
|------|------|--------|
| 用户查看入口 | 用户中心是否显示优惠券？ | P1 |
| 可用/已用/已过期 | 状态分类展示 | P1 |
