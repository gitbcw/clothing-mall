## 二、快速部署命令

### 2.1 完整部署（一键脚本）

```bash
# 在本地项目根目录执行
cd /c/MyFile/workspace/clothing-mall

# 1. 构建后端
mvn clean package -DskipTests

# 2. 构建前端
cd clothing-mall-admin && NODE_OPTIONS=--openssl-legacy-provider npm run build:dep && cd ..

# 3. 上传后端
scp clothing-mall-all/target/clothing-mall-all-0.1.0-exec.jar admin@47.107.151.70:/home/admin/clothing-mall/docker/litemall/litemall.jar

# 4. 上传前端
scp -r clothing-mall-admin/dist/* admin@47.107.151.70:/home/admin/clothing-mall/docker/admin-dist/

# 5. 重建并重启服务
ssh admin@47.107.151.70 "cd /home/admin/clothing-mall/docker && docker compose build app && docker compose up -d app"

# 6. 验证
curl -s http://47.107.151.70:8088/admin/auth/info
```

### 2.2 仅更新后端

```bash
# 本地打包
mvn clean package -DskipTests -pl clothing-mall-all -am

# 上传并重启
scp clothing-mall-all/target/clothing-mall-all-0.1.0-exec.jar admin@47.107.151.70:/home/admin/clothing-mall/docker/litemall/litemall.jar
ssh admin@47.107.151.70 "cd /home/admin/clothing-mall/docker && docker compose build app --no-cache && docker compose up -d app"
```

### 2.3 仅更新前端

```bash
# 本地打包
cd clothing-mall-admin && NODE_OPTIONS=--openssl-legacy-provider npm run build:dep

# 上传（Nginx 自动生效，无需重启）
scp -r dist/* admin@47.107.151.70:/home/admin/clothing-mall/docker/admin-dist/
```

### 2.4 仅更新数据库

```bash
# 从本地 Docker 导出
docker exec clothing-mall-mysql mysqldump -u clothing_mall -pclothing123456 clothing_mall --no-tablespaces --single-transaction > deploy/db/local_data.sql

# 上传并导入
scp deploy/db/local_data.sql admin@47.107.151.70:/tmp/
ssh admin@47.107.151.70 "docker exec -i clothing-mall-mysql mysql -u root -proot123456 -e 'DROP DATABASE IF EXISTS clothing_mall; CREATE DATABASE clothing_mall CHARACTER SET utf8mb4;' && docker exec -i clothing-mall-mysql mysql -u root -proot123456 clothing_mall < /tmp/local_data.sql"
```

## 五、常见问题

### 5.1 Maven 打包失败：文件被占用

**症状**：
```
Failed to delete .../target/xxx.jar
```

**解决**：停止占用进程
```bash
# 查找并停止 Java 进程
wmic process where "commandline like '%clothing-mall%'" get processid,commandline
taskkill /PID <PID> /F
```

### 5.2 前端构建失败：openssl-legacy-provider

**症状**：
```
ERR_OSSL_EVP_UNSUPPORTED
```

**解决**：
```bash
export NODE_OPTIONS=--openssl-legacy-provider
npm run build:dep
```

### 5.3 数据库导入失败：列数不匹配

**症状**：
```
ERROR 1136: Column count doesn't match value count at row 1
```

**原因**：本地和服务器的表结构版本不一致

**解决**：使用带列名的导出方式
```bash
mysqldump --complete-insert --skip-extended-insert ...
```

### 5.4 前端更新不生效

**原因**：浏览器缓存

**解决**：
1. 强制刷新：`Ctrl + Shift + R`
2. 或清除浏览器缓存
3. 或使用无痕模式

### 5.5 Docker 镜像拉取失败

**症状**：
```
Error response from daemon: Get "https://registry-1.docker.io/v2/": EOF
```

**解决**：使用国内镜像源或本地已有镜像
